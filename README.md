# ✊✌️✋ C# じゃんけんゲーム（学習用）

## 📌 概要
C# のオブジェクト指向設計を実践するために作成したコンソールじゃんけんゲームです。  
クラスを役割ごとに分離する設計を意識して実装しました。

## 🎯 作った目的
- `enum`（列挙型）の使い方を体験する
- クラスのプロパティ・コンストラクタを実際に使う
- 役割ごとにクラスを分ける設計（責務の分離）を身につける
- `Random` クラスによるランダム処理を理解する

## ✨ 機能一覧
- グー・チョキ・パーの3択でCPUと対戦
- CPUの手はランダムで自動生成
- 勝ち・負け・引き分けの判定と結果表示
- 連続対戦（何回でも続けられる）
- スコア表示（`s` キーで勝敗数を確認）
- 最終結果の表示

## 🛠️ 使用技術
- 言語：C# (.NET 10)
- 実行環境：コンソールアプリ（macOS）
- バージョン管理：Git / GitHub

## 📚 学習したこと
| 項目 | 内容 |
|------|------|
| enum | `Hand`（グー・チョキ・パー）と`Result`（勝敗）を定義 |
| プロパティ | `{ get; set; }` でデータの読み書き |
| コンストラクタ | `new Player("プレイヤー")` で初期設定 |
| クラス設計 | 役割ごとに4ファイルに分離 |
| Random | `_random.Next(0, 3)` でCPUの手を生成 |
| キャスト | `(Hand)randomNumber` で数値を型変換 |
| 三項演算子 | `条件 ? trueのとき : falseのとき` |
| インクリメント | `WinCount++` で勝ち数を加算 |

## 🗂️ ファイル構成
```
Janken/
├── Program.cs       # エントリーポイント（起動するだけ）
├── Hand.cs          # グー・チョキ・パーの定義（enum）
├── Player.cs        # プレイヤーの設計図（名前・スコア管理）
├── Judge.cs         # 勝敗判定ロジック
└── GameManager.cs   # ゲーム全体の進行管理
```

## 🚀 実行方法
```bash
# リポジトリをクローン
git clone https://github.com/ando625/Janken.git

# フォルダに移動
cd Janken

# 実行
dotnet run
```
